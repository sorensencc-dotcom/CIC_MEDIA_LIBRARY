# Continuous Improvement Analysis Skill

Reports token savings, bottleneck detection, skill gaps, phase velocity, permission hotspots.

## Usage

```bash
/improvement-analysis
/improvement-analysis days:14
/improvement-analysis days:7 detailed:true
```

## Features

- **Token Audit**: Model/provider distribution, cache effectiveness
- **Bottleneck Detection**: Repeated queries, permission blocks, context resets
- **Skill Gaps**: Manual tasks 3+ times → skill candidates
- **Phase Velocity**: Time/phase, blockers, critical path
- **Permission Hotspots**: Most-requested tools → allowlist candidates
- **Recommendations**: Ranked by impact (time/tokens/unblock)

## Implementation

**Location:** `C:\Users\soren\.claude\skills\improvement-analysis.js`

**Invocation:**

```bash
node ~/.claude/skills/improvement-analysis.js days:14
node ~/.claude/skills/improvement-analysis.js days:7 detailed:true
```

**Parses:**

- Session metadata: `~/.claude/sessions/*.json`
- Tool results: `~/.claude/projects/*/*/tool-results/*`
- Git history: `git log --since="2 weeks ago" --format='%H|%aI|%s|%b'`
- Permissions: `.claude/settings.json`, `settings.local.json`

**Detects:**

- Permission hotspots (frequency, cost rank)
- Bottlenecks (repeated queries, context resets, blocked tasks)
- Skill gaps (manual patterns 3+ occurrences)
- Phase velocity (time/phase, blockers, critical path)
- Token efficiency (cache hits, cost optimization)
